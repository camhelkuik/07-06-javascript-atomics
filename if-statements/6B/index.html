Did you use = or == or === to compare passwords (or something else)? Why? Would any of the others work?

I used === because the entered password will have to be exactly the same as the hardcoded one. I experimented with == and it seemed to work as well as ===. I prefer === because I understand that there is no "false" truth. A single = does not work because it is assigning the answer to equal the password so it will always be true.

Why is it a bad idea to store a password (or other sensitive information) in plain text in a Javascript file?

The sensitive information is not safe. Someone could purposfully or accidently come upon that information. It is easy to find.
