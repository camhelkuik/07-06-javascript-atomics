What's the difference between alert() and console.log()?

Alert opens up a pop up window that has the text and then lets you clost the window. Console.log is used for debugging. It logs or adds to the console when something is run.

What's one situation where you might use an `alert()` to tell the user something?

Alert would be good for an error to make sure that the user is aware of the error. It could be good to let the user know when they are about to exit a secure page.

What's one way that `alert()` might get obnoxious?

If you have an alert for every submission of information, or every action on a page has an alert.

Would it make sense to use console.log() to tell the user things? Why or why not?

It would not make sense because it only logs things to the console, which a user may not know how to get to and would most likely not think to look there for information.

What's one way that console.log() might be a good tool for you to use?

It would be helpful for debugging, to see if certain parts of the code was run.

When do the console.log() statements get run? Is it before the first alert() gets displayed? After it gets displayed but before the user clicks "OK" (or equivalent)? After the user clicks to close the dialog box? A different time? How did you find out for sure?
